# Security Policy

## Supported versions

GlossBot is still in early public development. Security fixes will be handled on the actively developed branch until stable releases exist.

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.

Use one of these private channels instead:

- GitHub private vulnerability reporting, if it is enabled for this repository
- a direct private message to the maintainer through GitHub

When reporting a vulnerability, please include:

- a clear description of the issue
- affected files or components
- reproduction steps or a proof of concept, if available
- impact assessment
- any suggested remediation

We will acknowledge valid reports as quickly as possible, investigate privately, and coordinate a fix before public disclosure when appropriate.
